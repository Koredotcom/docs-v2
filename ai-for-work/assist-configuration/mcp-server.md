# MCP Server 

The MCP Server feature enables users to manage Model Context Protocol (MCP) servers with agents as tools. This feature provides access to agents through the server for task execution and workflow management.
<img src="../images/mcp-server-home.png" alt="mcp-server-home" title="mcp-server-home" style="border: 1px solid gray; zoom:80%;"/>

## Default MCP Server

The platform includes a pre-configured Default MCP Server that simplifies initial setup and provides immediate access to core functionality. This server comes ready to use with comprehensive agent and knowledge source integration.

The platform includes a pre-configured Default MCP Server that contains:

* All pre-built agents are available as tools.
* Enterprise knowledge sources are integrated into the server.

Users can access this server without additional configuration, streamlining the initial setup process.


## Server Configuration

The Server Configuration section allows users to view and manage connection details for their MCP servers. Users can access essential information needed to integrate external applications with the platform.


### Accessing Server Details

To view the Default MCP Server configuration:

1. Navigate to the **MCP Server** page.
2. Select the **Default MCP Server** option.
3. The system displays a dialog box containing server details.
<img src="../images/mcp-server-url.png" alt="mcp-server-url" title="mcp-server-url" style="border: 1px solid gray; zoom:80%;"/>

Each MCP Server provides a unique URL that enables external applications and services to connect. Users can copy this URL directly from the interface for integration purposes.


## Key Features

* **Pre-built Agent Integration**: Access all platform agents as tools through a single server instance
* **Enterprise Knowledge Sources**: Leverage organizational knowledge bases within agent workflows
* **URL-based Access**: Connect external systems using the provided server URL
* **Copy Functionality**: Copy server URLs for configuration in other applications
