The **Monitoring** module in AI for Process tracks events, actions, and changes across your account. Use it to maintain security, ensure compliance, and optimize performance across roles, integrations, models, workflows, users, datasets, and guardrails.

| Feature | Description |
|---|---|
| [**Audit Logs**](/ai-for-process/settings/monitoring/audit-logs) | Provides full visibility into every user action and system event. Each log entry captures the event name, category, user, date and time, and a detailed description. Filter by time period, category, event type, or user, and export as CSV for compliance reporting. |
| [**Analytics**](/ai-for-process/settings/monitoring/analytics) | Monitor performance metrics and trends across workflows and models, including success rates, response times, and resource usage. |
| [**Model Traces**](/ai-for-process/settings/monitoring/model-traces) | Provides run-level analysis of any model deployed or integrated in your account, including request input, output, response time, deployment version, and source metadata. Filter by status, response time, source type, or request ID, and export as CSV. |
| [**Custom Scripts**](/ai-for-process/settings/monitoring/monitor-custom-scripts) | Provides run-level and log-level visibility into deployed custom scripts, including execution status, response time, function name, log level, and timestamp. Filter by time or column, or select any run to inspect its input, output, and log details. |
