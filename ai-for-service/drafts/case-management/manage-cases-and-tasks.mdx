## Manage Cases

A case is a formal record of a situation or problem that requires tracking, attention, and resolution. Managing cases involves a systematic approach to ensure that all aspects are addressed efficiently and effectively.

### Access Cases

Navigate to **Contact Center AI > Case Management > Manage Cases > Cases**.

![Cases](/ai-for-service/drafts/case-management/images/accessing-cases.png)

Each case includes the following details:

| Field | Description |
|---|---|
| **ID** | Unique identifier for the case or service request. |
| **Title** | Brief title describing the service request. |
| **Customer** | Name of the customer who requested the case. |
| **Channels** | Channel through which the request was made (Web or Mobile Client). |
| **Status** | Current state of the case. |
| **Priority** | Priority level: Critical, High, Medium, or Low. |
| **Last Updated** | Date and time of the last update. |
| **ETA** | Estimated due date. Displays as overdue, due today, or the target date. |
| **Assignee** | Agent assigned to the case (not applicable for Unassigned, My Cases, or Closed). |

Additional navigation options available:

| Option | Description |
|---|---|
| **My Cases** | Cases assigned to the current agent or back-office employee. |
| **My Tasks** | Tasks assigned to the current agent or back-office employee. |
| **Closed** | Closed cases. |
| **Search Cases** | Search by case ID, status, priority, or keywords in descriptions or notes. |
| **Download** | Downloads case management data in `.csv` format. |
| **Advanced Search** | Filter by Case ID, Case Title, Customer Email, Customer Phone Number, Description, or Requester. |

![Advanced Search](/ai-for-service/drafts/case-management/images/advanced-search-cases.png)

### Create a Case

1. Navigate to **Contact Center AI > Case Management > Manage Cases > Cases**. The unresolved cases page displays.  
   ![Create Cases](/ai-for-service/drafts/case-management/images/manage-cases-new-case.png)

2. Click **+ New Case**. The **New Case** dialog opens with a default case number assigned.

3. Complete the following fields (required fields are marked with a red asterisk):

   - **Requested by:** Enter the customer's name.
   - **Channel:** Select from the dropdown. For **Voice**, select the customer's phone number and country. For **Web/Mobile Client**, enter the customer's email address.
   - **Queue:** Select from the list.
   - **Case Type:** Select a case template. Templates are specific to the selected queue.
   - **Due Date:** Select a completion date.
   - **Description:** Enter a conversation summary of the issue or service request.
   - **Title:** Enter a case title based on the customer's issue.
   - **Priority:** Select **Critical**, **High**, **Medium**, or **Low**. The status defaults to **Open**.

4. Click **Save**. A success message confirms the case is created and assigned based on case type assignment rules.  
   ![New Case](/ai-for-service/drafts/case-management/images/new-cases-queues.png)

### View Cases

Case IDs and task IDs appear as links throughout the interface. Selecting a case ID link opens the case details in a new tab. You can view cases and tasks in separate tabs and return to the list view using the **Home** icon.

The **Conversations** section supports email communication. Create an email by selecting a recipient, queue, and template. All replies appear in the conversation thread. The conversation remains open until it times out or you manually close it.

<Note>To send an email, use the customized email template for that case.</Note>

![View Cases](/ai-for-service/drafts/case-management/images/view-cases-tile.png)

Available case views:

| View | Description |
|---|---|
| **Unresolved** | Active cases not yet resolved or closed. Click a case to view associated tasks, customer details, current status, and activity logs. |
| **Unassigned** | Cases identified but not yet assigned to a case manager or team member. |
| **All Cases** | All cases in the system, regardless of status. |
| **My Cases** | Cases assigned to the current agent or back-office employee. |
| **Closed** | Completed and closed cases. |
| **Search Cases** | Search by case ID, status, or priority. |

![Case Details](/ai-for-service/drafts/case-management/images/manage-task-details.png)

<Note>Supervisors can set a capacity limit for specific agents. Cases remain in queue until an agent is available.</Note>

### Transfer a Case

Select a new agent from the **Assignee** dropdown to transfer the case.

![Case Transfer](/ai-for-service/drafts/case-management/images/case-transfer.png)

### Resolve or Close a Case

1. In the case details, select **Resolve** from the status dropdown.  
   ![Resolve Case](/ai-for-service/drafts/case-management/images/resolved-case-update.png)

2. After completing the review process, view all case statuses — including **Closed**, **Open**, and **Resolved** — from **Dashboard > My Cases > Closed**.  
   ![Closed Cases](/ai-for-service/drafts/case-management/images/closed-my-cases.png)

3. To view the status of all closed cases on the dashboard and in Manage Cases, see [Analyze Cases](/ai-for-service/drafts/case-management/case-analysis-dashboard).  
   ![Managed Closed Cases](/ai-for-service/drafts/case-management/images/manage-all-cases-closed.png)

4. Ensure all required actions and activities associated with the case are completed and add a comment before closing.

5. Review the case to confirm all aspects of the customer's issue are addressed and resolved.

6. Mark all associated tasks as **Resolved**, then change the case status to **Closed**.

## Manage Tasks

Task management helps agents resolve cases efficiently by ensuring all required actions are completed. You can configure assignment rules for new cases to allocate tasks to agents or back-office employees based on their availability, capacity, and skills. The system automatically sends notifications when tasks are transferred or updated.

Agents can perform the following actions on assigned tasks:

- View and update all required fields (active or inactive).
- Add comments to track progress and provide context.
- Change the task status upon completion.
- Assign the task to themselves, or transfer it to another agent or back-office employee.
- View all task details, including previous comments and status changes.
- Create, edit, or delete tasks until the case is closed.

### Task Notifications

The system notifies agents through the console when new tasks are assigned.

![Task Notifications](/ai-for-service/drafts/case-management/images/task-notifications.png)

### Access Tasks

Navigate to **Contact Center AI > Case Management > Manage Cases > Tasks**.

![Tasks](/ai-for-service/drafts/case-management/images/manage-cases-tasks.png)

<Note>Agents can switch between custom tasks and task templates.</Note>

### Add a New Task

1. Click a case from the **Tasks** page.  
   ![Add New Task](/ai-for-service/drafts/case-management/images/add-new-task.png)

2. Click **+ Add New Task**. The **New Task** dialog opens with a default task number assigned.  
   ![New Task](/ai-for-service/drafts/case-management/images/new-task.png)

3. Complete the following fields (required fields are marked with a red asterisk):

   - **Task Type:** Select the task type from the dropdown.
   - **Due Date:** Select a completion date.
   - **Description:** Enter a summary based on the case conversation.
   - **Title:** Enter a title based on the issue raised.
   - **Priority:** Select **Critical**, **High**, **Medium**, or **Low**. The status defaults to **Open**.

4. Click **Save**. A success message confirms the task is created and assigned to the agent.

<Note>Agents cannot create or edit tasks after a case is closed. The system prevents task creation if required fields are missing or invalid.</Note>

### View Task Details

Click any existing unresolved task to view the task description, details, current status, and activity logs.

![View Tasks](/ai-for-service/drafts/case-management/images/view-tasks.png)

### Transfer a Task

Select a new agent from the **Assignee** dropdown to transfer the task.

When assigning a task:

- Match the task to an agent based on expertise, workload, and availability.
- Set the appropriate priority level to focus efforts where they are most needed.

### Review Task Activity

**To monitor tasks:**

1. Check the dashboard for task statuses, outstanding items, and overall workload.  
   ![Dashboard Monitor](/ai-for-service/drafts/case-management/images/task-monitor.png)

2. After tasks are completed, review comments and event details to track progress.

**To view task activity:**

1. In the **Activity** section, click the **Expand** icon.

2. Click the **Comments** tab to view or add comments.  
   ![Comments](/ai-for-service/drafts/case-management/images/view-activity.png)

3. Click **Save**. You can edit, delete, or add supported image files related to the task activity.  
   ![View Comments](/ai-for-service/drafts/case-management/images/resolved-issue.png)

4. Click the **Events** tab to view all events for a case, including status changes, updates, and actions taken by agents.  
   ![Events](/ai-for-service/drafts/case-management/images/activity-events.png)

After completing tasks, verify that all customer issues are resolved. Ensure that comments and events include appropriate details for each task — this information tracks communication progress and captures context throughout the case lifecycle for future reference.

### Update Task Status

In the **Task Details** section, select **Resolve** from the status dropdown.

![Update Task Status](/ai-for-service/drafts/case-management/images/closed-task.png)

### Close a Task

Change each task's status from **Resolved** to **Closed** after completing all required actions. After all review processes are complete, view task statuses — including **Closed**, **Open**, or **Resolved** — from **Dashboard > My Tasks > Closed**.

![Task Closure](/ai-for-service/drafts/case-management/images/my-tasks-closed.png)
