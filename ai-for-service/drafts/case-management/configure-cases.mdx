Administrators configure case management to match specific business needs using case templates, custom fields, task templates, statuses, and priorities.

## Navigation

Go to **Contact Center AI > Case Management > Configuration**.

## Key Components

| Component | Description |
|---|---|
| **[Case Templates](#case-templates-configuration)** | Pre-designed forms that help agents collect information for customer issues. |
| **[Fields](#fields)** | Custom data fields agents use to capture relevant information during case resolution. |
| **[Task Templates](#task-templates-configuration)** | Standardized task sets to streamline case management. |
| **[Status](#status)** | Case statuses to track progress throughout the lifecycle. |
| **[Priority](#priority-configuration)** | Priority levels to determine urgency and order for cases and tasks. |
| **[User Management](/ai-for-service/user-management/manage-users#manage-users)** | Add agents, assign roles, set capacity limits, configure skills, and manage queues. |

## Fields

Fields represent data agents collect for a case. The system includes default required fields, and you can create custom fields in different formats based on your business needs. Use fields to customize case templates.

### Access Fields

Navigate to **Contact Center AI > Case Management > Configuration > Fields**.

![Fields](/ai-for-service/drafts/case-management/images/config-fields.png)

### Add a New Field

1. On the **Fields** page, click **+ New Field**. The **General Settings** panel opens.  
   ![New Field](/ai-for-service/drafts/case-management/images/new-field-create.png)

2. Complete the **General Settings**:

   - Enter the **Display Name**.
   - Enter the **Name** for internal reference.
   - Enter a **Description**.
   - Select the **Field Format** from the dropdown.
   - Enter the **Placeholder** text.
   - Enable **Read-only** if applicable.
   - Enter **Tooltip Text** to display on hover.
   - Click **Save**.  
     ![General Settings](/ai-for-service/drafts/case-management/images/general-settings.png)

3. In **Data Settings**, enter the **Default Value** based on the field format, and click **Save**.  
   ![Data Settings](/ai-for-service/drafts/case-management/images/data-settings.png)

   <Note>For the **Number** field format, enter the value and press **Enter**.</Note>

4. In **Validation Settings**, complete the following and click **Save**:

   - Enter the **Default Error Message**.
   - Select the validation condition from the **Validate** dropdown.
   - Select the **Validation Rule** and enter a rule value. Optionally, enter a **Custom Error Message**.

   <Note>You can add any number of validation rules.</Note>

   ![Validation Settings](/ai-for-service/drafts/case-management/images/validation-settings.png)

### Edit a Field

<Note>Only custom fields are editable. System fields cannot be edited.</Note>

1. On the **Fields** page, click the three-dot menu icon for the custom field.  
   ![Edit Fields](/ai-for-service/drafts/case-management/images/edit-fields.png)
2. Click **Edit**.  
   ![Edit Field Page](/ai-for-service/drafts/case-management/images/edit-field-page.png)
3. Modify **General Settings**, **Data Settings**, or **Validation Settings** as needed.
4. Click **Save**.

### Delete a Field

<Note>Deleting a field affects all case templates currently using it.</Note>

1. On the **Fields** page, click the three-dot menu icon for the field.  
   ![Delete Fields](/ai-for-service/drafts/case-management/images/delete-fields.png)
2. Click **Delete** to confirm.  
   ![Delete Confirmation](/ai-for-service/drafts/case-management/images/delete-confirm.png)

## Task Templates Configuration

Task templates provide a predefined layout that guides agents when creating or managing tasks. You can customize default assignments, add fields, and configure assignment and escalation rules to ensure efficient task allocation upon case creation.

### Access Task Templates

Navigate to **Contact Center AI > Case Management > Configuration > Task Templates**.

The **Task Templates** page displays:

| Column | Description |
|---|---|
| **Name** | Task template name. |
| **Description** | Purpose of the task. |
| **Created by** | Administrator who created the template. |
| **Status** | Current template status. |
| **Active** | Whether the template is active or inactive. |

Use the **Search** field to find templates by name.

![Task Templates](/ai-for-service/drafts/case-management/images/task-templates.png)

### Create a New Task Template

1. On the **Task Templates** page, click **+ New Task Template**.

2. Complete the **Basic Information** section:

   - Enter a **Name**.
   - Enter a **Description**.
   - Choose the **Default Assignee**:

     | Option | Description |
     |---|---|
     | **Unassigned** | Creates tasks without assigning them to any individual. |
     | **Queues** | Assigns the task to a custom queue (for example, contact center agents or back-office agents). |
     | **Individual Agent** | Assigns to a specific agent automatically. |
     | **Task Creator** | Assigns the task to the person who created it. |

   ![New Task Template Basic Info](/ai-for-service/drafts/case-management/images/new-task-template-basic-info.png)

   <Note>Every task template must have a default assignee: a queue, an individual agent, or an agent group.</Note>

3. Click **Next** to proceed to the **Add Fields** page.

4. Click **+ Add Fields** to add custom fields to the template. Mark fields as required or optional.  
   ![Custom Fields](/ai-for-service/drafts/case-management/images/edit-add-fields.png)

5. Click **Create** to finalize the template.  
   ![Create](/ai-for-service/drafts/case-management/images/add-fields-create.png)

### Edit a Task Template

1. On the **Task Templates** page, click the three-dot menu icon for the template.  
   ![Edit Task Template](/ai-for-service/drafts/case-management/images/edit-task-template.png)
2. Click **Edit** to modify the basic information.  
   ![Edit Basic Info](/ai-for-service/drafts/case-management/images/edit-basic-info.png)
3. Modify existing fields or add new ones as needed.  
   ![Modify Fields](/ai-for-service/drafts/case-management/images/edit-add-fields.png)
4. Click **Update**.

### Delete a Task Template

On the **Task Templates** page, click the three-dot menu icon for the template, and click **Delete**.

![Delete Task Template](/ai-for-service/drafts/case-management/images/delete-task-template.png)

<Note>Only administrators can deactivate cases and restrict agent access to case types. You cannot delete a case if it has associated templates currently in use.</Note>

## Case Templates Configuration

Case templates are pre-designed forms that help agents gather and organize information for various customer issues. Templates include a name, description, default assignment, default fields, and optional custom fields and predefined tasks.

### Access Case Templates

Navigate to **Contact Center AI > Case Management > Configuration > Case Templates**.

The **Case Templates** page displays:

| Column | Description |
|---|---|
| **Name** | Unique case template name. |
| **Description** | Description of the customer issues covered. |
| **Created by** | Administrator who created the template. |
| **Status** | Draft or published status. |
| **Active** | Whether the template is active. When an active template is deactivated, agents can no longer use it for new cases, but it remains in the system for future use. |

Use the **Search** field to find templates by name.

![Case Templates](/ai-for-service/drafts/case-management/images/case-templates.png)

### Create a New Case Template

1. On the **Case Templates** page, click **+ New Case Template** and complete the **Basic Information** section.  
   ![New Case Template](/ai-for-service/drafts/case-management/images/case-templates-creation.png)

   a. Enter a **Name** for the case type.

   b. Define the **Case ID**.

   <Note>You cannot delete the Case ID after it is created.</Note>

   c. Enter a **Description** for the template's purpose.

   d. Choose the **Default Assignee**:

   | Option | Description |
   |---|---|
   | **Unassigned** | The case is not automatically assigned upon creation. |
   | **Queues** | Assigns to a queue for a specific agent group. Configure a limit of 15 cases per agent — cases exceeding the limit remain in queue. |
   | **Individual Agent** | Automatically assigns to a specific agent. |
   | **Case Creator** | Assigns to the user who created the case. |

   ![Case Fields](/ai-for-service/drafts/case-management/images/create-a-new-case-template.png)

2. Click **Next** to proceed to the **Case Fields** page. Case fields ensure agents capture specific information for each case. Mark newly added fields as required or optional.  
   ![Case Fields](/ai-for-service/drafts/case-management/images/add-case-fields.png)

3. Click **+ Add Fields** and complete the following:

   | Field | Description |
   |---|---|
   | **Field Name** | For example, Customer Name or Issue Description. |
   | **Type** | Agent-created (**Custom**) or admin-defined (**System**). |
   | **Validation** | Mark as required or optional. |
   | **Action** | Delete the field if not required. |

4. Click **Next** to proceed to the **Task** page.  
   ![Task Page](/ai-for-service/drafts/case-management/images/new-case-next.png)

5. On the **Task** page:

   - Enable the **Custom Task** toggle to allow agents to create custom tasks within the case.
   - Click **+ Add Task**, select a **Task Name** from the dropdown, and click **Create**.  
     ![Add Task](/ai-for-service/drafts/case-management/images/new-case-task.png)

### Edit a Case Template

1. On the **Case Templates** page, click the three-dot menu icon for the template.  
   ![Edit Case Template](/ai-for-service/drafts/case-management/images/edit-case-temlate.png)
2. Click **Edit**, modify the **Basic Information** (Name and Description), and click **Next**.  
   ![Modify Basic Info](/ai-for-service/drafts/case-management/images/edit-basic-info-case.png)
3. Modify **Case Fields** as needed, and click **Next**.  
   ![Edit Case Fields](/ai-for-service/drafts/case-management/images/case-fields-edit.png)
4. Modify **Task** fields as needed, and click **Update**.  
   ![Modify Task](/ai-for-service/drafts/case-management/images/edit-task-case.png)

### Delete a Case Template

On the **Case Templates** page, click the three-dot menu icon for the template, and click **Delete** to confirm.

![Delete Case Template](/ai-for-service/drafts/case-management/images/delete-case-confirm.png)

## Status

Status configuration tracks the progress and state of cases and tasks. The system provides five default statuses — **Open**, **In Progress**, **On Hold**, **Resolved**, and **Closed** — and you can create custom statuses to meet specific business requirements.

| Status Type | Description |
|---|---|
| **System Statuses** | Predefined; cannot be edited. |
| **Custom Statuses** | Created by administrators; can be edited, activated, deactivated, or deleted. |

### Access Status

Navigate to **Contact Center AI > Case Management > Configuration > Status**.

![Status Page](/ai-for-service/drafts/case-management/images/new-status-access.png)

### Manage Existing Statuses

System statuses cannot be edited. You can activate or deactivate both system and custom statuses using the toggle switch. Inactive statuses become unavailable elsewhere in the system.

### Add a New Status

1. On the **Status** page, click **+ New Status**.

2. Complete the following:

   - Enter a **Name** for the status.  
     ![New Status Fields](/ai-for-service/drafts/case-management/images/new-status-fields.png)
   - Select a **Status Type** from the dropdown: **Open**, **In Progress**, **On Hold**, **Resolved**, or **Closed**.
   - Add a **Description** for context (optional).
   - Enable the **Status Settings** toggle to make the status available to agents.

3. Click **Save**. New statuses are **Active** by default.

<Tip>
- Create specific statuses for internal use — for example, **Pending Customer Response** instead of a generic **On Hold**.
- Consider stage-specific statuses — for example, **Reopened** for resolved cases that need further attention.
</Tip>

### Edit a Status

<Note>Only custom statuses can be edited.</Note>

1. On the **Status** page, click the three-dot menu icon for the status.  
   ![Edit Status](/ai-for-service/drafts/case-management/images/edit-status-edit.png)
2. Click **Edit**. The **Edit Status** panel opens.  
   ![Edit Status Fields](/ai-for-service/drafts/case-management/images/edit-status-fields.png)
3. Enable the **Enabled** toggle to make the status available to agents.
4. Click **Save**.

### Delete a Status

1. On the **Status** page, click the three-dot menu icon for the status.  
   ![Delete Status Menu](/ai-for-service/drafts/case-management/images/edit-status-delete.png)
2. Click **Delete** to confirm.  
   ![Delete Status](/ai-for-service/drafts/case-management/images/delete-status.png)

## Priority Configuration

Priority configuration determines which cases and tasks require immediate attention. The system includes four default priorities — **Low**, **Medium**, **High**, and **Critical** — and administrators can create custom priorities for specific business requirements.

| Priority Type | Description |
|---|---|
| **System Priorities** | Predefined; cannot be edited. |
| **Custom Priorities** | Created by administrators; can be edited or deleted. |

### Access Priority

Navigate to **Contact Center AI > Case Management > Configuration > Priority**.

![Priority Page](/ai-for-service/drafts/case-management/images/new-priority-access.png)

### Manage Existing Priorities

System priorities cannot be edited, but you can deactivate them using the toggle switch. The **Priority** page lists all priorities sorted by their assigned value.

### Add a New Priority

1. On the **Priority** page, click **+ New Priority**.

2. Complete the following:

   - Enter a **Name**.  
     ![New Priority](/ai-for-service/drafts/case-management/images/new-priority-enabled.png)
   - Add a **Description** for context.
   - Enter a **Value** (1–15) reflecting the priority numerically.
   - Select a **Color** as a visual indicator.
   - Enable the **Priority Status** toggle to make it available to agents.

3. Click **Save**. The new priority appears on the **Priority** page and is **Active** by default.

<Tip>
- Use a consistent numbering system — for example, increments of 5 or 10 — to allow easy insertion of new priorities later.
- Choose distinct colors for each priority level for quick visual identification.
- Consider your business needs when creating custom priorities — for example, an **Emergency** level above **Critical**.
</Tip>

### Edit a Priority

<Note>Only custom priorities can be edited.</Note>

1. On the **Priority** page, click the three-dot menu icon for the priority.  
   ![New Priority](/ai-for-service/drafts/case-management/images/new-priority-enabled.png)
2. Click **Edit**.  
   ![Edit Priority](/ai-for-service/drafts/case-management/images/edit-priority.png)
3. Modify the required fields and click **Save**.

### Delete a Priority

1. On the **Priority** page, click the three-dot menu icon for the priority.  
   ![Delete Priority](/ai-for-service/drafts/case-management/images/delete-priority.png)
2. Click **Delete**.  
   ![Delete Custom Priority](/ai-for-service/drafts/case-management/images/delete-custom-priority.png)
