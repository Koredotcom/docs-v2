The Agent Console displays essential information for agents using the Case Management Console. It supports initiating customer interactions, creating cases, and managing cases and tasks.

## Prerequisites

Before managing cases and tasks in the console:

- Set up a channel and a chat or voice AI Agent with customer support dialogs.
- Ensure customers have the option to select an agent during AI Agent or voice interactions.
- The customer must select the agent transfer option to initiate agent interaction.

## Create a Case

Cases are created based on the configuration defined in Case Templates. You can create cases using two methods:

| Method | Description |
|---|---|
| **[Automated](#automated)** | Configured using public APIs for channels such as emails and voicemails. |
| **[On-Demand](#on-demand)** | Created during a live conversation based on the agent's review of the customer's issue. |

### Automated

Automated case creation uses public APIs for channels such as emails and voicemails. Use these APIs to retrieve, update, and delete case details.

<Note>When automated case creation is enabled, every qualified conversation automatically generates case data, and a case is created.</Note>

### On-Demand

On-demand case creation applies to live conversations via email, chat, or voice. Agents and supervisors review the customer's issue and create a case accordingly.

Agents and supervisors can create on-demand cases through:

- Offline Conversation
- Live Conversation

<Note>Administrators can enable on-demand case creation in the **Manage Cases** section and assign it to specific agents instead of enabling it for all agents.</Note>

## Offline Conversation

Agents and supervisors can manually create cases for interactions that aren't linked to a real-time conversation — such as emails, voice calls, or chat messages. This doesn't require an active conversation with the customer, enabling agents to manage customer issues and inquiries outside of live interactions.

Create a case using one of the following:

- [Agent Console](#create-a-new-case)
- [Manage Cases and Tasks](/ai-for-service/drafts/case-management/manage-cases-and-tasks#create-cases-outside-the-agent-console)

### Create a New Case

1. Navigate to **Contact Center AI > Agent Console**.  
   ![Agent Console](/ai-for-service/drafts/case-management/images/console-page.png)

2. Click the **+** icon, then click **Create Case**. The **New Case** dialog opens with a case number assigned.  
   ![Create Case](/ai-for-service/drafts/case-management/images/manage-new-case-unassigned.png)

3. In **Requested by**, enter the name of the customer requesting the case — for an issue, inquiry, complaint, or service request.

4. Select a channel from the **Channels** dropdown:

   - **Voice:** Select the phone number and country for the customer.
   - **Web/Mobile Client:** Enter the customer's email address.

5. Select a case template from the **Case Type** dropdown and complete the form with the customer's issue details.

6. Enter a **Title** based on the customer's issue.

7. Enter a **Description** based on the conversation summary.

8. Select a **Due Date**.

9. Select the **Priority** (**Critical**, **High**, **Medium**, or **Low**) from the dropdown. The **Status** defaults to **Open**.

10. Select any appropriate options in the **Sample** field to support the new case.

11. Click **Save**. A success message confirms the case is created.

<Note>The case is assigned to agents based on the case type assignment rules.</Note>

## Live Conversation

Agents create and track cases during real-time interactions via email, voice, or chat, enabling them to address customer issues while the conversation is ongoing.

### Agent Transfer

When a customer requests an agent, the system initiates a transfer workflow.

![AI Agent Transfer](/ai-for-service/drafts/case-management/images/ai-agent-transfer.png)

- **Notification Alert:** The agent receives a notification in the agent console when a conversation is transferred. [Learn more](https://docs.kore.ai/xo/console/managing-incoming-interactions/).  
  ![Agent Transfer Notification](/ai-for-service/drafts/case-management/images/agent-transfer.png)

- **Identify the Issue:** The agent reviews the conversation history to identify customer issues or inquiries. Because customers may have multiple issues, agents can use the search function to locate specific ones.

  Case IDs and task IDs appear as links throughout the interface. Selecting a case ID link opens the case details in a new tab. You can view cases and tasks in separate tabs and return to the list view using the **Home** icon. The system supports up to 10 open tabs for cases and tasks combined.

  ![Issues Search](/ai-for-service/drafts/case-management/images/issues-search.png)

- **Follow-up Questions:** To further troubleshoot, the agent gathers additional details from the customer, including:

  - Troubleshooting information: brand name, model number, part details, and images of defective parts.
  - Supporting assets: [attachments](https://docs.kore.ai/xo/console/interacting-with-customers/?h=attachment#sending-attachments-to-customers), [agent forms](/ai-for-service/contactcenter/configurations/agent-forms/configure-agent-forms), and [hyperlinks](/ai-for-service/console/interacting-with-customers#adddelete-hyperlinks-in-the-text-editor) (web or email addresses).
  - New case creation, if additional follow-up is required.  
    ![Create New Case](/ai-for-service/drafts/case-management/images/create-new-case.png)

  <Tip>Based on the conversation, the agent provides relevant information or suggests actions to resolve the issue — for example, troubleshooting steps for a reported refrigerator cooling issue using the customer details received.</Tip>

### Case Creation via Live Interaction Pane

1. Click the **New Case** icon.  
   ![New Case Icon](/ai-for-service/drafts/case-management/images/new-case-icon.png)  
   The **New Case** dialog opens with a case number assigned.

2. Select a case template from the **Case Type** dropdown and complete the form with the customer's issue details.

3. Enter a **Title** based on the customer's issue.

4. Enter a **Description** based on the conversation summary.

5. Select a **Due Date**.

6. Select the **Priority** (**Critical**, **High**, **Medium**, or **Low**) from the dropdown. The **Status** defaults to **Open**.

7. Select any appropriate options in the **Sample** field.

8. Click **Save**. A success message confirms the case is created and assigned to an agent.  
   ![New Case Number](/ai-for-service/drafts/case-management/images/new-case-number.png)

### Case Notification Alert

- **Agent notification:** The system notifies the agent as soon as a case is assigned to them.
- **Notification history:** Click the bell icon to view all previous notifications.

### Case Creation During a Voice Call

1. Click the **New Case** icon on the voice call screen to open a case form that includes the call transcript and relevant customer information.  
   ![Voice Call Screen](/ai-for-service/drafts/case-management/images/voice-call.png)

2. Select a case template from the **Case Type** dropdown. The template displays with **Assigned to**, **Title**, and **Description** pre-filled.  
   ![Case Template](/ai-for-service/drafts/case-management/images/new-case-template.png)

3. Select the **Priority** (**Critical**, **High**, **Medium**, or **Low**). The **Status** defaults to **Open**.

4. From the **Account Type** dropdown, select the type of account associated with the case.

5. From the **Product Type** dropdown, select the product or service category related to the case.

6. Click **Save**.

### Case Creation via Email

1. Navigate to **Contact Center AI > Flows & Channels > Digital**.  
   ![Digital Navigation](/ai-for-service/drafts/case-management/images/digital-navigation.png)

2. Select **Email** to create a new email domain.  
   ![Domain Email](/ai-for-service/drafts/case-management/images/digital-email.png)

3. Click **New Email Address** to register an email address for the domain. [Learn more](/ai-for-service/channels/add-email-channel#create-a-kore-domain-email-address).  
   ![Domain Email](/ai-for-service/drafts/case-management/images/domain-email.png)

4. Attach a **Flow Experience** to your custom or Kore domain email address.

5. Publish the email address. [Learn more](/ai-for-service/channels/add-email-channel#publish-kore-domain-email-addresses).

6. Compose an email from your mailbox with the required information and click **Send**. The agent receives an email transfer acceptance response and initiates the customer interaction.  
   ![Customer Interaction](/ai-for-service/drafts/case-management/images/customer-interaction.png)

7. Click **Reply** to gather information from the customer about the issue.  
   ![Customer Reply](/ai-for-service/drafts/case-management/images/customer-reply.png)

8. Click the **New Case** icon to register a new case or service request. [Learn more](#create-a-new-case).  
   ![New Case from Email](/ai-for-service/drafts/case-management/images/transfer-call.png)

9. Use the **Transfer** option to forward the email conversation to another agent.  
   ![Call Transfer](/ai-for-service/drafts/case-management/images/call-transfer.png)

10. Click **End** to close the conversation after registering or addressing the issue.
