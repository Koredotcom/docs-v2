Follow these steps to set up case management in AI for Service, enable features, and validate your configuration.

## Step 1: Sign Up

Log in to the AI for Service (XO11) registration page and complete the sign-up form with your email address. After registration, the platform creates your workspace with default configurations. [Learn more](/ai-for-service/getting-started/navigating-the-platform).

## Step 2: Enable Case Management

1. After registration, note your unique **Account ID** — required to activate case management.
2. Verify that you have admin access (granted automatically after registration).
3. Contact the Kore Product team for activation assistance at [support.kore.ai](https://support.kore.ai/).

## Step 3: Configure Queues and Skills

- **Queue Setup:** Create and customize queues for different teams — such as contact center agents and back-office staff. [Learn more](/ai-for-service/contactcenter/routing/queues/queue-management).
- **Default or Custom:** Use default queues or build custom ones as needed.
- **Skills:** Configure agent skills for proper case routing (optional). [Learn more](/ai-for-service/user-management/manage-users#skills).
- **Hours of Operation:** Adjust to match your business schedule — for example, weekday hours or 24/7. [Learn more](/ai-for-service/contactcenter/configurations/hours-of-operation/manage-hours-of-operation#hours-of-operation).

## Step 4: Add Channels

Configure your communication channels (Email, Chat, Voice). [Learn more](/ai-for-service/channels/adding-channels-to-your-bot).

## Step 5: Invite Team Members

1. Send email invitations to agents and supervisors.
2. Assign appropriate roles, queues, and skills. [Learn more](/ai-for-service/user-management/manage-users#queues-skills).
3. Enable capacity limits per agent as needed.

## Step 6: Set Up Case Management Components

Configure the following before agents begin using case management:

- **[Fields](/ai-for-service/drafts/case-management/configure-cases#fields):** Create custom fields for your business needs. Available formats:
  - Drop-down lists
  - Multi-select options
  - Checkboxes
  - Phone numbers
  - Email addresses
  - URLs

- **[Status](/ai-for-service/drafts/case-management/configure-cases#status):** Create custom statuses to supplement the five default statuses.

- **[Priority](/ai-for-service/drafts/case-management/configure-cases#priority-configuration):** Create custom priorities with numeric values to supplement the four default priorities.

- **[Task Templates](/ai-for-service/drafts/case-management/configure-cases#task-templates-configuration):** Create templates to manage and automate customer support tasks. Choose the default assignee:
  - Unassigned
  - Queue
  - Individual agent
  - Task creator

- **Additional Fields:** Add any custom fields required for each task template.

- **[Case Templates](/ai-for-service/drafts/case-management/configure-cases#case-templates-configuration):** Create templates to help agents collect information for customer issues. Include:
  - Custom fields (name, description, default assignment)
  - Predefined tasks (unassigned, queue, individual agent, or case creator)

## Step 7: Set Up Case Layout

Design an efficient case layout for the agent console. [Learn more](/ai-for-service/console/manage-layout#adding-new-layout).

## Step 8: Set Up User Permissions

Configure the following access controls:

- [Manage Users](/ai-for-service/user-management/manage-users#manage-users)
- [User Attributes](/ai-for-service/user-management/manage-users#attributes)

## Step 9: Configure Case Handling Capacity and Language Settings

1. Go to **User Management > Manage Users**.  
   ![Manage Users](/ai-for-service/drafts/case-management/images/user-mgt.png)

2. Select a user, then click the three-dot menu icon and select **Edit**.  
   ![Edit User](/ai-for-service/drafts/case-management/images/manage-users-edit.png)

3. On the **User Attributes** page, select **Cases**.  
   ![User Attributes](/ai-for-service/drafts/case-management/images/attribute-edit-cases.png)

4. Turn on the **Enable Case** toggle to configure the **Maximum Case Handling Capacity** for the agent.

   <Note>The maximum capacity for both **Case** and **Task** is 15 each.</Note>

5. Under **Language Support**, select the desired language.

6. Select the **Proficiency Level** for the selected language.

7. Click **Update Changes**.

## Step 10: Test Your Setup

Test your configuration using the following scenarios.

### Email Channel

- Verify that all case management components work correctly from case creation through resolution.
- Verify that case management integrates with your email system.
- Set up an email account for sending and receiving test cases.
- Prepare templates for case creation and communication.

### Automation Process

1. Create an automated flow.
2. Configure failover handling. When automation fails, the system routes the conversation to an agent in the default queue (Agent 1 and Agent 2).
3. Create a new queue named **Case Queue** and add Agent 3 and Agent 4.
4. Create a [case template](/ai-for-service/drafts/case-management/configure-cases#case-templates-configuration) with **Case Queue** as the default assignee.

### Conversation Process

1. **Send Email:** The system processes the conversation through the default queue flow.
2. **Automation Failure:** When automation fails, the system routes the conversation to Agent 1 or Agent 2.
   - **Case Creation:** When Agent 1 creates a case, the system assigns it to Agent 3 or Agent 4. [Learn more](/ai-for-service/drafts/case-management/agent-console#create-a-new-case).
   - **Case Handling:** Agent 1 can close the conversation while the case remains open for the assigned agent to continue.
